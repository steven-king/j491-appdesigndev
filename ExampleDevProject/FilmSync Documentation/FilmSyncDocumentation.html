1. API Description and Scope

Mobile application development for Android and iOS uses the SDK/Static Library/Plugin. The respective library is included within your own application and the following events and methods are available.

Second screen content can be produced via Wordpress with the FilmSync plugin installed or any other CMS that can generate JSON data. Using the Wordpress Plugin, the content producer creates and orders the content within Wordpress admin tool. Specific tones that correspond with specific chapters and cards are embedded in the source video.

Definitions:

Tone

18,000hz - 19,000hz for .5 seconds

Signal

        A combination of two tones for a total of 1 second.

Marker ID

        The numerical id of a signal.

2. App SDK Documentation

Available for Android and iOS

Settings

audioSourceID

        Unique identifier that relates directly to specific content source. Required.

contentSourceURL

        Parent URL to source of content. Typically wordpress blog url (ex. http://docsinhand.com)

timeOut

        The specified time the app will continue to listen for source tones. Default, 2 minutes.

Events

listeningForSource

        Dispatched when app is ready to find source audio

sourceDetected

Dispatched once when a signal from an external source is detected for a specific source ID. Will not be dispatched till source is lost.

sourceLost

        Dispatched once when no tone detected in timeOut duration.

markerDetected

        Dispatched when a signal is detected. Sends currentMarker id.

Methods

getTotalMarkers

        Returns the number of chapters to be delivered during the entire duration of the source content (via JSON count of objects). Number

getCurrentMarker

        Returns the numerical id of current marker number detected from video source. Starts with 0. Returns null if no source detected.

getPreviousMarker.

        Returns the numerical id of previous marker number detected from the source content. This may not be in numerical order as user can move throughout the video source. If first marker, returns null.

getTonesDetected

        Returns array of hzs of the last detected signal. Length max of 2.) ex. [18200, 19400]

3. WordPress Plugin Documentation

Provides an admin user interface via Wordpress plugin/theme for creating content delivered via REST API to App

Content producers can use Wordpress to create second screen content. A video source can dispatch multiple chapters and cards. Cards are organized without chapters. Each chapter and each card has a unique identifier tone embeded in the video source.

Translate WordPress Taxomony to App language

Posts = Cards

Categories = Chapters

Provide Easy interface for ordering Posts/Chapters rather than dependent on Date/Time.

Provides Plugin Config

        sourceID text field

        twitter search (hash tag or handle) text field

        Source URL (text field)

        About (text field)
