FilmSync App Technology

Task: 

Develop an iOS and Android SDK that can be used for internal and external development and create respective demo applications using the respective SDKs.

FilmSync App is a second screen experience, which provides contextual information during a documentary film. The prototype was built using Phonegap and now we need to build out the technology in a way that will work for applications beyond video such as conference speaking, live events, etc.

FilmSync is based on pitch detection and audio watermarking technology.

Technology:

The current prototype detects a single tone in the 18,000-19,000 hz range which limits the number of event. This project requires detection of a specific frequency for example 18,800hz and reads a 32bit integer from a rapid pulse of the tone providing about 4 million unique tones. Essentially, a virtual modem of ultrasonic tones.

Data will use dotted quadd nottion 130.5.5.25 and the tone will be based on 10000010.00000101.00000101.00011001

First two groups will represent the client’s unique id. Third grouping will be the project or film id and the last grouping will be the unique card or event.

There should also be a way to generate this 32 bit tone (audio file) so it can be added to the source material (film audio track) Phase two will be a web interface for generating tones and cards.

SDK Description and Scope

Mobile application development for Android and iOS uses the SDK/Static Library/Plugin. The respective library is included within your own application and the following events and methods are available.

Second screen content is delivered via JSON data. Example below.

Specific audio tones that correspond with specific chapters are embedded in the source video.

Definitions:

StarterTone

18,800hz for .5 seconds. This is used to alert the device a 32bit id is coming.

Signal

        A combination of starter tone and pulsed tone with 32bit number.

Marker

        A specific audio file embedded in the video source to note an event. This is what defines the time when a card should be delivered

Card

        Visual display of content associated with each tone that is provided by the JSON. For example, a single film will have many cards.

App SDK Requirements

Available for Android and iOS

Settings

sourceID

        Unique identifier that relates directly to specific content source. Required. (licensing number)

contentSourceURL

        Parent URL to source of content SON file (ex. http://docsinhand.com/api/12345.json)

timeOut

        The specified time the app will continue to listen for source tones. Default, 2 minutes.

Events

listeningForSource

        Dispatched when app is ready to find source audio

sourceDetected

Dispatched once when a signal from an external source is detected for a specific sourceID. Will not be dispatched again till source is lost.

sourceLost

        Dispatched once when no signal detected in timeOut duration.

markerDetected

        Dispatched when a proper signal is detected. Sends currentCard id.

Methods

getTotalCards

        Returns the number of cards to be delivered during the entire duration of the source content (via JSON count of objects). Number

getCurrentCard

        Returns the numerical id of current marker number detected from video source. A 32bit number . Returns null if no source detected.

getPreviousCard.

        Returns the numerical id of previous marker number detected from the source content. This may not be in numerical order as user can move throughout the video source. If first marker, returns null.

getTonesDetected

        Returns array of hzs of the last detected signal. Length max of 2.) ex. [18200, 19400]

getSignalDetected

        Returns the ID of the signal detected

Web Admin Interface with JSON API

Provides functionality for users to create an account with basic information such as name, company, email, etc. (See requirements document Section 3)

A user will then create new “projects” (aka Films)

A project will consist of “Cards” which are triggered by the embedded audio events. (See requirments section 3)

A user will be able to click a button that generates a zip file of all the cards necessary for the project.

User can edit content of a card and no change to audio files is necessary.

When user saves a project, generates a JSON file for a CDN.

Example Use Cases:

Independent FilmMaker

Signup with web interface.

Filmmaker creates a project( film) in web interface

Filmmaker create their content (multiple cards) for each event. For example 12 events/cards

Filmmaker clicks downlonload markers and can download of the 12 unique audio files that includes the numbers embedded in the file with the filenames marker01.aiff or marker02.mp3 in a zip file.

Filmmaker then adds the files to the video source material.

Filmmaker clicks “Send to App” button to add it to the FilmSync app.

FilmSync Admins will add the URL to the  list of projects/films which populates the JSON file of the list of films that populate FilmSync.

Video source now works with FilmSync app for that film.

Commercial Film Study that Licenses the code for use

Company signs up for an account.

Company licenses the code through the web interface

Access to the two SDKs are provided to developers.

Company creates a project in web interface

Developers implement SDK into native apps.

Company creates content (multiple cards) for each event.

Company clicks downlonload markers and can download of the 12 unique audio files that includes the numbers embedded in the file with the filenames marker01.aiff or marker02.mp3 in a zip file.

Company then adds the files to the video source material.

Company clicks “Send to App” which provides the url to the JSON file created to be added to the app.
